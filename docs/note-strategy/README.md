# 「家庭内稟議」note × X 運用プロジェクト

「妻に上手いこと言って、月の飲み会の回数を増やす方法」を起点にした、note有料記事 × X集客の副業プロジェクトの戦略・運用ルール一式。

## ドキュメント構成

| ファイル | 内容 | 用途 |
|---|---|---|
| [strategy.md](./strategy.md) | 市場分析・ポジショニング・価格戦略・収益計画 | 意思決定の土台。四半期ごとに見直す |
| [persona.md](./persona.md) | 発信者の人格設定・口調・ディテールバンク | **Claudeに毎回渡すコンテキスト** |
| [note-writing-rules.md](./note-writing-rules.md) | note記事の構成・描写・課金ラインのルール | **note執筆時にClaudeに渡すコンテキスト** |
| [x-operation-rules.md](./x-operation-rules.md) | X運用ルール・投稿の型・ローンチ手順・炎上予防 | **X投稿生成時にClaudeに渡すコンテキスト** |

## Claudeでの運用フロー

### note記事を書くとき
1. `persona.md` + `note-writing-rules.md` をコンテキストに渡す
2. 「◯◯というテーマでnote記事の構成案を出して」→ 構成を人間が確定
3. セクションごとに下書き生成
4. **人間の必須作業**: `persona.md` のディテールバンクから実体験を注入する。AI量産型noteとの差別化はここが全て。実話の固有名詞・金額・セリフが入っていない下書きは公開しない
5. 「AI臭チェックリスト」（note-writing-rules.md 末尾）を通して推敲

### X投稿を作るとき
1. `persona.md` + `x-operation-rules.md` をコンテキストに渡す
2. 週次バッチで「来週分のポストを型別に20本」生成
3. 人間が7〜10本に選別し、実体験ディテールを1箇所以上足して予約投稿
4. 反応の良かったポストを翌週の生成プロンプトに例として貼る（勝ちパターンの学習）

### 週次レビュー（30分）
- X: インプレッション上位3ポストと下位3ポストを比較 → 勝ち型を特定
- note: ビュー→購入率を確認（目安1〜3%）
- `strategy.md` のKPIセクションに数字を追記

## 最重要原則（全ドキュメント共通）

1. **妻は攻略対象ではなく、交渉相手として最大限のリスペクトを払う相手**。倒す敵は「気まずさ」「無計画」「先延ばし」であって妻ではない
2. **嘘をつくテクニックは書かない・売らない**。「上手いこと言う」= 騙すではなく「妻が気持ちよくYESと言える提案を設計する」こと
3. **具体描写が商品価値**。金額・時刻・セリフの再現がないコンテンツは出さない
