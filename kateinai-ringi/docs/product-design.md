# 商品設計 — 根幹noteとパターンnoteの二層アーキテクチャ

## 1. 設計思想: 教科書と問題集

| | 根幹note | パターンnote |
|---|---|---|
| 売るもの | **体系**（フレームワーク＋自分史） | **適用**（特定シチュエーションへの実装・症例） |
| 例えるなら | 教科書 | 問題集・過去問 |
| 本数 | **最大3本**（下記。増やさないことが価値） | 無限に刊行できる |
| 価格 | 980〜1,480円（値上げで育てる） | 500〜680円（衝動買い圏） |
| 更新戦略 | **育てるnote**。追記・Q&A追加→値上げ | **刊行するnote**。vol.を重ねる・季節で出す |
| 分量 | 有料部分 8,000〜12,000字 | 有料部分 3,000〜6,000字 |
| 役割 | 利益の柱・アカウントの看板 | 入口商品・根幹への送客・市場調査 |

### なぜ根幹は3本で止めるか

1. **体系が4つ以上ある専門家は信用されない**。「稟議の人」という記号が薄まる
2. 根幹同士が食い合わないためには、**軸が直交している**必要がある。直交する軸は「時間・金・土壌」の3つで打ち止め（下記）
3. ネタの量産はパターン側の仕事。「妻は一人」問題（strategy.md 9.）の解でもある: 自分史を消費するのは根幹だけにして、パターンは相談案件・類型適用で書く

### 二層の相互送客

- パターンnote巻末に「この処方箋の背景にある体系は根幹noteに全部書いた」→ **上りのファネル**
- 根幹note巻末に「あなたの状況特化の実装はパターンnoteへ」→ **下りのクロスセル**
- どのパターンが売れるかが、次の根幹・次のパターンの市場調査になる（先にパターンで小さく検証→反応が大きい類型を厚く展開）

## 2. 背骨: 稟議五類型（全noteで共通利用するフレームワーク）

商品群を貫く診断体系。**名前をつけて全コンテンツで同じ用語を使う**（これ自体がアカウントのIPになり、読者が自分の症状を自己診断→該当noteを買う導線になる）。

| 類型 | 症状 | 正典の症例 |
|---|---|---|
| **①未提出型** | 否決が怖くて申請自体を出せない。「言い出せない」 | （相談案件に最多と予想） |
| **②白紙申請型** | 「行っていい?」だけ。日時・帰宅時刻・予算・メンバーの情報がない | 「で、何時に帰るの」案件 |
| **③手続き違反型** | 事後報告・直前申請。内容以前に手続きで死んでいる | 冷戦事件、「私は聞いてないよね」 |
| **④予算超過型** | 頻度・金額が家計・家庭リソースの現実と衝突している | 忘年会シーズン、小遣い問題 |
| **⑤信頼残高不足型** | 申請は正しいのに通らない。過去の実績・平時の行いが原因 | F-3花束事件、S-3義母バーター |

- X投稿・相談回答・note本文すべてで「これは③手続き違反型ですね」と使う
- 診断チャート（5問で自分の類型が分かる）は**無料で配る**（無料note・根幹noteの無料部分）。診断は撒き餌、処方箋が商品

## 3. 根幹note（3本・軸で直交させる）

### K-1『月1回だった飲み会が、月4回になった。— 家庭内稟議の通し方【完全版】』
**軸: 時間の稟議** ｜ 本命・最初に出す ｜ 680円（先着50部）→980円→1,280円

```
【無料部分】
1. 月4回飲みに行く男の家の、日曜夜の風景（結果を先に見せる）
2. 原点: 4,200円と3週間の冷戦（正典4を完全再現）
3. 「言えばいいじゃん」と独身の後輩は言う（悩みの代弁）
4. なぜ会社の稟議は通るのに、家庭の稟議は通らないのか（構造分析）
5. 稟議五類型と診断チャート（★ここまで無料で出し切る）
6. 目次公開・価格の理由・返金案内
――――― 課金ライン ―――――
【有料部分】
7. 原則編: 家庭内稟議5原則（事前申請・根回し・代替案・予算明示・事後報告）
8. 月初一括申請制の完全手順（S-1。この記事の主力商品）
9. シチュエーション別セリフ台本12本（急な誘い/金曜連チャン/土曜昼から/出張明け…）
10. 死亡フラグワードNG集10（F-1「みんな行くから」ほか、なぜ死ぬかの解説付き）
11. タイミング設計 — 決裁者カレンダー理論（F-4レセプト事件）
12. 事後フォロー — 飲み会は翌朝の上履きで終わる（F-5）
13. ケーススタディ: ある金曜の完全ログ（Slackの誘い17:00→承認→帰宅→翌朝まで時系列再現）
14. 巻末: 稟議前チェックリスト10項目（保存用）
```

### K-2『小遣い35,000円の男が、家庭内予算交渉で学んだ全て — 金の稟議編』
**軸: 金の稟議**（小遣い値上げ・高い買い物）｜ K-1の3〜4ヶ月後 ｜ 980円→1,480円

- 調査で判明した**完全空白市場**（夫側の小遣い値上げ交渉noteは存在しない）
- 正典の2019年小遣い交渉（30,000→35,000円、A4一枚の資料）が原点エピソード
- 時間の稟議との違い: 単発承認ではなく**恒久予算の改定**なので、根拠資料・段階提案・家計への還元設計が必要 — より「実用系」に寄るため価格を上げられる（note公式データ: 実用系の売れ筋平均は1,800円）
- 高い買い物（ゴルフクラブ・PC・時計）の一時金稟議も同軸でカバー

### K-3『稟議が通る家庭は、申請の前に何をしているか — 平時運用編』
**軸: 土壌**（信頼残高・関係資本の平時づくり）｜ 最後発。ファン層確認後 ｜ 980円

- ⑤信頼残高不足型への処方箋。S-3義母バーター、F-3花束事件が中核
- **注意: 説諭系（競合最大勢力）に最も接近する危険地帯**。「妻を理解しよう」ではなく「決裁が速い組織の平時運用」というビジネスフレームを最後まで貫くこと
- 出すのは月商3万円を超え、タカシ節が確立してから。早く出すと「結局説教かよ」でブランドが死ぬ

> K-4以降は作らない。新しい体系を思いついたら、まず既存根幹への「追記」かパターンnoteで検証する。

## 4. パターンnote（刊行ライン）

フォーマット: 相談要約 or 状況設定 → 五類型で診断 → 処方箋（セリフ台本中心）→ 根幹noteへの参照。1本3,000〜6,000字・500〜680円。

| # | タイトル方向 | 中身 | 発売タイミング |
|---|---|---|---|
| P-1 | 忘年会シーズン攻略（12月は特別会計） | S-4の完全版。5回分を一括で通す期初交渉、12月の死亡フラグ | **11月中旬固定**（季節商品。3月歓送迎会版に改題再販可） |
| P-2 | 稟議事例集 vol.1 — 通らない稟議10案件を診断する | 相談案件10本の診断＋処方箋。note限定の深掘り3本入り | 実相談が10件たまり次第。以降vol.を定期刊行 |
| P-3 | ゴルフ・丸一日系の大型稟議 | 「夜」ではなく「休日を丸ごと」使う稟議は別競技。家族の休日設計とのバーター | K-1の反応を見て |
| P-4 | 泊まり系（出張明け・キャンプ・旅行） | 連泊は④予算超過型の最難関。前後の家庭内実績設計 | 相談の集まり次第 |
| P-5 | 推し活・趣味の高額買い物編 | K-2の姉妹。一時金稟議のセリフ台本 | K-2発売後のクロスセル |
| P-6 | 乳幼児期の特別ルール編 | 最難関市場（そもそも申請凍結期をどう生きるか）。悩みが最も深い層 | ③観察事例＋相談が揃ってから。炎上チェック最厳格で |

- パターンnoteが3本たまったらマガジン『家庭内稟議研究所 紀要』にバンドル（単品合計の6〜7割）
- **どのPが売れたかを次の投資判断に使う**（例: P-5が伸びる→K-2を前倒し・厚くする）

## 5. 無料note（ファネルの中間層・忘れがちな要件）

Xの140字と有料noteの間に「無料の試食note」を1本置く。固定ポストのリプ欄から常時導線。

- タイトル案: 『妻の「別にいいけど」翻訳講座 — 家庭内稟議・入門』
- 中身: 妻のセリフ集（正典7）の翻訳＋五類型の簡易診断。最後にK-1へ
- 役割: スキ・フォロー稼ぎ、note内検索からの流入口、K-1無料部分への滑走路
- K-1発売の1〜2週間前に公開する

## 7. このファイルの運用

- 新しいnoteの企画はまずここに追記（根幹に足したくなったら「本当にパターンで検証できないか」を自問する）
- 各noteの実売数・価格変更履歴をこのファイルの表に追記していく
