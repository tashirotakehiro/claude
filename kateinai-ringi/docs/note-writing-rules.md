# note執筆ルール

note記事をClaudeで書く際、`persona.md` と一緒にコンテキストとして渡す。

## 1. 記事の基本構成（本命記事・姉妹記事共通）

```
【無料部分】全体の30〜40%
1. フック         — 結果を先に見せる1〜3行（「月1だった飲み会が月4になった。夫婦仲は前より良い」）
2. 失敗談         — 原点の失敗をフルディテールで描写（ここが試食の目玉。出し惜しみしない）
3. 悩みの代弁     — 読者の頭の中のセリフをそのまま書く（「言えばいいじゃん、と独身の同僚は言う」）
4. 構造分析       — なぜ通らないのか。「会社の稟議は通るのに家庭の稟議が通らない」の種明かし
5. 目次公開       — 有料部分の目次を全部見せる
6. 価格の理由＋返金設定の案内
――――――――― 課金ライン ―――――――――
【有料部分】8,000〜12,000字目安
7. 原則編         — 家庭内稟議の5原則（事前申請・根回し・代替案・予算明示・事後報告）
8. 実践編         — シチュエーション別セリフ台本（そのまま使える台詞をカギ括弧で10本以上）
9. NG集           — 死亡フラグワード集（「みんな行くから」「たまにはいいだろ」等、なぜダメかの解説付き）
10. タイミング設計 — いつ言うか（月初申請の技術、妻の繁忙日を避ける、給料日との関係）
11. 事後フォロー   — 飲み会は帰宅で終わらない（翌朝の動き方まで）
12. ケーススタディ — 実際の交渉の一部始終を1本、時系列で完全再現
13. チェックリスト — 印刷して使える「稟議前チェック10項目」
```

### 課金ラインの原則
- **無料＝「なぜ」、有料＝「どうやって」**。構造分析まで無料で出し切り、「じゃあ具体的に何と言うか」の直前で切る
- 無料部分だけで1本のエッセイとして成立させる（スキ・拡散は無料部分で稼ぐ）
- 導入だけ無料のAI量産型と真逆を行く。無料部分の質が広告

## 2. 描写ルール（差別化の核心）

1. **数字を必ず入れる**。「遅く帰った」→「終電を逃してタクシー4,200円」。「少し不機嫌」→「返信が『わかった』の4文字になった」
2. **セリフはカギ括弧で一言一句再現**し、言い方の描写を添える。「『別にいいけど』——この『けど』が第二稟議の始まりである」
3. **LINEのやりとりは画面再現形式**で書く（送信時刻つき）
4. **五感で書く**。玄関を開けた瞬間のリビングの照明、食洗機の音だけがする夜、子どもの寝息
5. **妻側の内心を推測するパートを必ず入れる**。「妻はこう思っていたはずだ」→後日の答え合わせ。これが「妻が読んでも面白い」を担保する
6. 抽象論を2文続けたら、3文目は必ず具体例
7. エピソード・数字・セリフは `canon.md` の正典からのみ引く。正典にない過去をその場で発明しない。新エピソードが必要なら【正典追加候補】として別出しし、canon.md に登録してから使う

## 3. タイトルルール

- 32字以内を目安。数字＋ビフォーアフターを優先
- 妻を主語にした攻撃的表現を避ける（×「妻を説得する方法」→ ○「家庭内稟議の通し方」）
- 候補は毎回5本出して選ぶ

例:
- 「月1回だった飲み会が月4回になって、夫婦仲がむしろ良くなった話」
- 「会社の稟議は通るのに、家庭の稟議が通らない人へ」
- 「『別にいいけど』を本当の『いいよ』に変える、家庭内稟議の技術」

## 4. 冒頭1画面のルール

スマホ1画面（約200字）で「結果」と「これは説教noteではない」ことが伝わること。反省を促す気配を出したら読者（＝説教に疲れた夫）は即離脱する。

## 5. 有料部分の品質基準

- 「そのまま使えるセリフ」が10本以上あるか
- 読者が今夜から実行できるアクションが3つ以上あるか
- 各テクニックに「なぜ効くか」の一段深い理由があるか（妻側の視点での合理性）
- 失敗パターン（NG集）に自分の実話が最低1つ紐づいているか

## 6. 公開前チェックリスト

- [ ] `persona.md` のNGワードが含まれていない
- [ ] 「妻が読んだら不快になる箇所」がゼロ（迷ったら削る）
- [ ] 嘘・ごまかしを推奨する記述がない
- [ ] エピソード・数字・セリフが全て `canon.md` の正典と一致している（発明された過去がない）
- [ ] 検証可能な実績（部数・売上・感想・「実際に来た相談」）の捏造がない
- [ ] 数字・セリフの再現が無料部分に3箇所以上ある
- [ ] 返金設定ON、目次公開済み
- [ ] AI臭チェック（下記）通過

### AI臭チェックリスト
- [ ] 「〜ではないでしょうか」「いかがでしたか」が無い
- [ ] 箇条書きが3連続していない（本文は物語で運ぶ）
- [ ] 「重要なのは」「ポイントは」の乱発が無い
- [ ] 同じ語尾が3文続いていない
- [ ] 一般論だけの段落が無い（全段落にタカシの体験か視点が入っている)

## 7. 発売後の運用

- 購入者の感想は（許可を得て）記事末尾に追記していく → 値上げの根拠になる
- 読者からの質問はQ&Aとして追記 → 「買った後も育つnote」として値上げ告知の材料にする
- 値上げ時は必ずXで事前告知（「◯日で980円→1,280円」）— 駆け込み購入が発生する
