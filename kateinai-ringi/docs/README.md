# 「家庭内稟議」note × X 運用プロジェクト

「妻に上手いこと言って、月の飲み会の回数を増やす方法」を起点にした、note有料記事 × X集客の副業プロジェクトの戦略・運用ルール一式。

## Claudeでの運用フロー

### note記事を書くとき
1. `persona.md` + `canon.md` + `note-writing-rules.md` をコンテキストに渡す
2. 「◯◯というテーマでnote記事の構成案を出して」→ 構成を人間が確定
3. セクションごとに下書き生成
4. **人間の必須作業**: エピソード・数字・セリフが全て `canon.md` の正典と一致しているか照合する。正典にない過去が発明されていたら削るか、正典に登録してから使う。AI量産型noteとの差別化はこの一貫性が全て
5. 「AI臭チェックリスト」（note-writing-rules.md 末尾）を通して推敲し、`canon.md` の使用ログに記録

### X投稿を作るとき
1. `persona.md` + `canon.md` + `x-operation-rules.md` をコンテキストに渡す
2. 週次バッチで「来週分のポストを型別に20本」生成
3. 人間が7〜10本に選別し、正典との整合を確認して予約投稿。使用ログを更新
4. 反応の良かったポストを翌週の生成プロンプトに例として貼る（勝ちパターンの学習）

### 新しいエピソードが必要になったとき
`canon.md` の「新規エピソードの追加手順」に従う（候補5本生成→1本採用→正典登録→それから使用）。自分史の新規追加は月2本まで。ネタの主力は相談案件・観察事例（strategy.md 9.）へ

### 週次レビュー（30分）
- X: インプレッション上位3ポストと下位3ポストを比較 → 勝ち型を特定
- note: ビュー→購入率を確認（目安1〜3%）
- `strategy.md` のKPIセクションに数字を追記

## 最重要原則（全ドキュメント共通）

1. **妻は攻略対象ではなく、交渉相手として最大限のリスペクトを払う相手**。倒す敵は「気まずさ」「無計画」「先延ばし」であって妻ではない
2. **嘘をつくテクニックは書かない・売らない**。「上手いこと言う」= 騙すではなく「妻が気持ちよくYESと言える提案を設計する」こと
3. **具体描写が商品価値**。金額・時刻・セリフの再現がないコンテンツは出さない
4. **エピソードは創作してよいが、実績は捏造しない**。販売部数・売上・購入者の感想・「実際に来た相談」など検証可能な事実は絶対に作らない（strategy.md 9. 誠実性ガードレール）。キャラクター運用と虚偽広告は別物
