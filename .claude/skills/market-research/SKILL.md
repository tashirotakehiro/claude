# マーケ調査チーム（判定ファースト）

以下の商品についてマーケ調査を実施してください。
まず `.claude/skills/_shared/business-model.md` と `.claude/skills/_shared/notion-rules.md` を読むこと。

**商品情報:** $ARGUMENTS

**構造上の原則: 先に判定、後に成果物。キャッチコピーやLP構成といった「Goが前提の成果物」を判定より先に作ってはならない（作ると判定がGoに引っ張られる）。判定はキル基準に対する証拠で行い、雰囲気で総合判断しない。**

## 出力スタイル（チャット報告・Notion書き込みとも厳守）
- 全セクション**箇条書き**で書く。長い段落・文章での説明は禁止
- 1項目1行を基本とし、どうしても必要な場合のみ2行まで
- 「結論 → 根拠」の順。前置き・言い回しの装飾は削る
- 比較・一覧は表を使う（競合、価格、シナリオ等）

実行前に必ず:
1. この企画の**ドシエ**（notion-rules.md参照）を読む。アンカー定義・ジャッジメントテスト判定（軸×判定×確信度）・検証課題をチャット要約ではなくドシエから取得する
2. notion-rules.md のルールに従いフィードバックログの直近30件を読み、過去の没/見送りパターンを判定基準に反映する

## フェーズA: Go/No-Go判定

### 再審ルール（ジャッジメントテストとの役割分担。二重裁判をしない）
- ジャッジメントテストの軸×判定×確信度を**初期値として引き継ぐ**。確信度「高」の判定は、フェーズAで**新しい証拠**が出た場合のみ覆せる（＝再審）。同じ証拠で蒸し返さない
- 調査リソースの優先順位: ①グレー軸（検証課題）の解消 ②確信度「中/低」の軸の定量化 ③フェーズA固有の項目（上限価格・ターゲット量・ポジショニング・ペインポイント）
- 再審が発生した場合は「旧判定 → 新証拠 → 新判定」をドシエのジャッジメントテスト記録に追記する

3つのサブエージェント（Agentツール）を**並列で**起動する。
**起動時の必須事項**: サブエージェントはリーダーのコンテキストを共有しない。各エージェントのプロンプトに、ドシエから取得した**ジャッジメントテスト判定（軸×判定×確信度）**と**検証課題（グレー軸）**を必ず埋め込むこと。これを渡さないとエージェント3が攻撃対象を絞れず、際限なく調査して返ってこなくなる。

### エージェント1: 需要検証担当
- どんな悩み・課題を解決するか（統計データは**出典URL必須**。見つからなければ「未確認」と明記。捏造厳禁）
- ターゲット定義（性別・年齢・職業・ライフスタイル）と、ターゲット量の推測（検索ボリューム・市場データに加え、**市場実測シグナル**＝競合のレビュー増加速度→推定月販、広告の継続配信期間。business-model.md参照）
- **上限価格（顧客が買う可能性のある最大値）の推定**: business-model.md の価格設定原則に従い、①代替手段の累計コスト ②競合の最高価格帯 ③悩みの深さ の3点から算出。「売りやすい無難な価格」に丸めない
- **価格の根拠は必須**: 「なぜこの価格でいけると考えるか」を、上記3点それぞれの実データ（金額・商品名・出典）付きの箇条書きで添える。根拠のない価格は書かない。根拠が弱い場合は価格レンジと「どのデータが足りないか」を明示

### エージェント2: 競合・レビュー分析担当
- **ベネフィット競合**を含む競合8商品以上（サプリ・アプリ・医療・施術など別カテゴリも。同じ悩みを解決する商品は全て競合）
- 各競合の価格・機能・レビュー評価・低評価レビューの不満点
- 主要競合の**レビュー増加速度→推定月間販売数**と、広告ライブラリでの**配信継続期間**（＝実際に売れている・広告が回っている証拠。机上の市場規模データより優先）
- ポジショニング分析、市場ギャップ、勝ち筋
- 顧客ペインポイントTOP5と、LP訴求に使える感情的キーワード集

### エージェント3: 悪魔の代弁者（No-Go専任）
- **この商品が売れない理由だけ**を証拠付きで構築する。Goの材料は一切探さない
- ただし再審ルールに従い、**ジャッジメントテストが確信度「高」で通過させた論点を同じ証拠で蒸し返さない**。攻めるのは ①グレー軸 ②確信度「中/低」の軸 ③ジャッジメントテストが見ていない新しい事実

**終了条件（これを守らないと収束せず返ってこない。厳守）:**
- **出力は反証3〜5本まで**（No-Goに効く強い順）。6本目以降は探さない。「もっと強い反証があるか」を無限に探さない
- **攻撃対象を固定する**: business-model.md の共通キル基準5項目 ＋ 起動プロンプトで渡されたグレー軸・確信度「中/低」の軸のみ。それ以外は対象外
- **1論点あたりWeb検索は1〜2回まで**。出典が見つからなければ「未確認（出典なし）」と明記して次へ進む（＝“売れない証明”を出典付きで固めようと沼らない。悪魔の証明を避ける）
- **出力フォーマット**（1反証1行の表）: `反証 | 該当キル基準/軸 | 証拠(URL or 未確認) | 深刻度(高/中/低)`
- 最後に「**最も致命的な反証1つ**」を明示して終える

### リーダー裁定（あなたの仕事）
1. 3エージェントの結果を統合し、**統合サマリー**（商品概要・差別化ポイント・上限価格とその根拠・懸念点）を書く
2. **キル基準チェック表**を作る（ジャッジメントテストの判定を初期値とし、フェーズAで更新された軸だけ差分「旧判定 → 新証拠 → 新判定」を明示。更新のない軸は再掲のみ）
3. 判定: **Go / 条件付きGo（条件を明記）/ No-Go**
4. 判定には必ず「**確信度**（高/中/低）」と「**どんな証拠が出ればこの判定が覆るか**」を添える
5. この時点でユーザーに判定を報告する

## フェーズB: 成果物制作（Go / 条件付きGo の場合のみ）
1. この商品カテゴリの選び方（重要な仕様要素、★★★〜★で重要度）
2. バナーキャッチコピー案（5案以上、各30文字以内。直球/好奇心/Before After/数字/共感のタイプ別）
3. LPのFVとざっくり構成案
4. LP内容から逆算した商品仕様（必須★★★ / 試行錯誤★★ / あれば嬉しい★）
5. 収益試算（楽観/基本/悲観の3シナリオ。販売価格はフェーズAの**上限価格を基準**にし、価格を下げた場合の感度も添える。年間3,000万円の達成条件を明示。CPC/CVRの仮定は市場実測シグナルで裏を取り、**裏の取れない定数だけの試算は「未検証」と明記**）

**No-Goの場合はフェーズBを作らない。** 代わりに「何を変えれば再挑戦できるか（ピボット方向）」を3行で示して終了する。

## Notion書き込み
- ドシエ（商品ページ）の下に「マーケ調査」サブページを作成
- フェーズA完了時点で判定までを書き込み、フェーズB完了後に追記する
- 調査で解消・更新された**検証課題**は、ドシエの検証課題セクションを更新する
- notion-rules.md の分割書き込みルールを厳守（1回3セクション程度まで、3〜5回に分割）

## フィードバックログ
判定確定後、notion-rules.md のルールに従いログへ1行追記する。ユーザーが判定に異を唱えた場合（AIはGoだがユーザーは見送り等）は、ユーザーの判定と理由を記録する — これが最重要の学習データ。
