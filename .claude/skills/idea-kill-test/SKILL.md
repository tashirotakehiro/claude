# キルテスト担当（アイデアを殺しに行く）

以下の商品アイデアを**殺せるかどうか**検証してください。
まず `.claude/skills/_shared/business-model.md` と `.claude/skills/_shared/notion-rules.md` を読むこと。

**商品アイデア:** $ARGUMENTS

**スタンス: あなたの仕事はアイデアを磨くことではなく、殺すこと。デフォルト判定はNo-Go。証拠付きの反証が成立した軸だけ「通過」とする。褒めるための調査はしない。ここで死なないアイデアだけが、マーケ調査のコストをかける価値を持つ。**

## 6つのキル軸（各軸で実際に調査すること。想像で書かない）
1. **価格競合**: Amazon/楽天で実際に検索する。¥3,000以下の類似品が上位にあるか。あるなら、その低評価レビューに攻め筋（明確な不満）があるか
2. **法規制**: 薬機法・景表法で、核となるベネフィット訴求（フック・FVコピー）がそのまま表現できるか。できない場合、迂回表現でフックの強度が保てるか
3. **経済性**: 原価率30%で製造/仕入れは現実的か。サイズ・重量的に物流コストは成立するか。悲観シナリオ（CPC高め × CVR 1%）でCPAが粗利内に収まるか
4. **空白の理由**: なぜこの商品はまだ存在しないのか。過去に似た商品が出て消えた形跡がないかWeb検索する（「作れるのに無い」ものには大抵理由がある）
5. **本音/建前ギャップ**: 広告をタップする瞬間・購入する瞬間・商品が家に届く瞬間に、心理的な恥ずかしさはないか。建前ラッパーは機能しているか
6. **ターゲット量**: この悩みに¥10,000払う人は年間3,000万円分いるか（検索ボリューム・市場データで概算）

## 出力

### 1. キルレポート
各軸について:
- **キル論拠**: このアイデアが死ぬ理由（本気で殺しに行く）
- **反証**: 証拠付きでのみ可。証拠がなければ「反証不能」と書く

### 2. 判定
- **生存**: 6軸すべてで反証成立
- **条件付き生存**: 1軸だけ弱いが、企画の修正で回避可能（修正案を明記）
- **没**: 反証不能の軸が残る（どの軸で死んだかを明記し、ピボットの方向性があれば3行で示す）

### 3. 生存・条件付き生存の場合のみ: プロジェクト概要
Notion商品ページに記載する内容。各項目2〜5行:
- **商品名（仮）**
- **開発コンセプト**（何を目的とした商品なのか。一言＋補足）
- **テーマの本質**（着地アンカーの本音・建前と、その証拠）
- **主な機能・性能**（どのように目的を達成するのか。納得の理屈を含む）
- **顧客のメリット**（「〇〇によって△△できる」形式）
- **想定ターゲット**（「〇〇の悩みを持った人」形式）
- **競合商品**（3商品。URL・価格・機能・差別化ポイント）
- **仕入価格（推定）**（原価率30%から逆算）
- **販売価格（推定）**（顧客が買う可能性のある最大値。business-model.md の価格設定原則に従い、「なぜこの価格でいけるのか」の根拠＝代替コスト・競合上限・悩みの深さの実データを必ずセットで書く。無難な価格に丸めない）

### 4. フィードバックログ追記
判定確定後、notion-rules.md のルールに従いログへ1行追記する。
