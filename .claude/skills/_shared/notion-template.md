# Notion 企画テンプレート契約（商品企画スキル共通）

商品企画の成果物は、プロジェクトDBの正式ページテンプレート **「【新商品】企画段階定型」** を複製して埋める。このファイルはその「型」と「どのスキルの出力をどの欄に入れるか」の対応表・手順を定義する単一仕様書。idea-judgment / market-research / product-planning / notion-rules はこれを参照すること。

## 保存先とID
- **プロジェクトDB（データソース）**: `collection://174b9edd-0d82-815d-a28c-000b1b069e30`（名称「プロジェクト」／くDB配下「商品企画管理」内）
- **複製元テンプレート**: 「【新商品】企画段階定型」`20db9edd0d82806fb451e1a6dc188ebe`
- タイトルプロパティ名は **`プロジェクト名`**、ステータスは **`ステータス`**（status型）
- 作成時に設定するプロパティ: **`分類`**（select＝新商品）／**`ステータス`**（新商品ｱｲﾃﾞｱ・固定）／**`提案者`**（person＝田代剛大）／**`企画作成日`**（date＝当日）。詳細は「作成・書き込み手順」2.を参照

## 全体構造（＝企画マスターページ）
1つの企画＝プロジェクトDBの1ページ。テンプレの並びは以下（この順・この見出し文言・トグル構造を壊さない）:

```
### 商品名（仮）：
▸ 商品案(初期)                         … トグル
**1.開発コンセプト(…)：**
**2.主な機能・性能(…)：**
**3.顧客のメリット(…)：**
**4.想定ターゲット(…)：**
▸ 競合商品(…)：                        … トグル（商品1〜3：URL/価格/機能/差別化）
**5.仕入価格(倉庫入荷まで含めた価格)：**
**6.販売価格：**
  └─（ここに「ドシエ」サブページを置く）
▸ マーケ調査                            … トグル（1〜13）
▸ 確定した商品概要                      … トグル（後工程で確定。デモ内容は削除）
目次 / 議事録
```

- **企画概要 = 1〜6**（＋商品名・商品案・競合商品トグル）
- **マーケ調査 = マーケ調査トグル内の 1〜13**
- **ドシエ**（内部の作業メモ＝アンカー/フック/ジャッジメント記録/検証課題）は「6.販売価格」の直下にサブページとして格納する

## 作成・書き込み手順
1. **作成（テンプレを"行"として生成）**: `notion-create-pages` で作る。
   - `parent` は必ず **`{type:"data_source_id", data_source_id:"174b9edd-0d82-815d-a28c-000b1b069e30"}`**（＝プロジェクトDBの行として作成される）
   - `pages[0].template_id` に **`20db9edd0d82806fb451e1a6dc188ebe`**（テンプレ「【新商品】企画段階定型」）を指定し、テンプレのレイアウト（見出し・トグル・表）を適用する。`content` は渡さない（template_id が本文を供給する）
   - `pages[0].properties.プロジェクト名` に企画名を入れる
   - ⚠️ **`notion-duplicate-page` は使わない**。テンプレページを duplicate するとコピーも「DBのテンプレート」として作られ（Notion UIに「テンプレートを編集しています」と出る）、**DBの行一覧に現れない**。必ず上記の create-pages + template_id で"行"として作ること。
   - template_id の適用は**非同期**なので、直後は本文が空のことがある。`notion-fetch` で本文（見出し・`<empty-block/>`）が揃ってから書き込む。
2. **プロパティ設定**: `notion-update-page` の `update_properties` で以下を必ず設定する（複製直後に1回）:
   - `プロジェクト名` ＝ 企画名
   - `分類` ＝ `新商品`（select）
   - `ステータス` ＝ `新商品ｱｲﾃﾞｱ`（status型・半角カナ）。**マーケ調査が終わってもこのステータスのまま変更しない**（以前の「マーケ調査着手時にマーケ企画中へ更新」ルールは廃止）
   - `提案者` ＝ 田代剛大（person型。値はユーザーIDのJSON配列 `["<user_id>"]`。IDは `notion-search`（query_type=user）で `tashiro.takehiro@survaq.com` を引く。現行ID: `eb081f10-9914-4ee2-b852-ce678af80065`）
   - `企画作成日` ＝ 当日（`date:企画作成日:start` にYYYY-MM-DD、`date:企画作成日:is_datetime` は 0）
   - ※`date:マーケ調査開始日:start` は任意で設定してよい。
3. **本文の埋め込み**: `notion-update-page` の `update_content`（search-replace）で、各見出し直下の `<empty-block/>` を実内容に置換する。
   - **注意**: テンプレには同一の `<empty-block/>` が多数ある。`old_str` は必ず**直前の見出し文言ごと**含めて一意にする（見出しなしで `<empty-block/>` 単体を置換しない）。トグル内はタブインデントが付くので、fetch した実文字列をそのままコピーして使う。
   - タイムアウト防止のため notion-rules.md の分割ルール（1回3セクション程度・3〜5回）を守る。トグル/表/目次などブロック構造は壊さない。
4. **ドシエ格納**: ドシエは企画マスターページの**子ページ**として作る（`notion-create-pages` の parent=マスターページ page_id）。位置は「6.販売価格」の直下（＝マーケ調査トグルの手前）に置く。ドシエの中身（アンカー定義/フック・FV/ジャッジメント記録/検証課題）は notion-rules.md のドシエ定義に従う。

## 出力スタイル
- 各欄は箇条書き中心・結論→根拠（notion-rules / 各スキルの出力スタイルに従う）。
- テンプレのMarkdown（見出しレベル・トグル・改行・「└」記法）に沿わせる。前置きの装飾文は入れない。
