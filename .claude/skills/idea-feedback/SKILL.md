# フィードバック記録係（企画精度を複利で上げる）

まず `.claude/skills/_shared/notion-rules.md` を読むこと（ログの場所と形式はそこに定義されている）。

**入力:** $ARGUMENTS

## モード判定
- 入力に判定・理由が含まれる → **記録モード**
- 「振り返り」「傾向」「まとめ」「分析」等 → **分析モード**

## 記録モード
1. 入力から「テーマ / 案名 / ステージ / 判定 / 理由」を抽出する。不明な項目は直前の会話やNotionの商品ページから補完し、理由が無ければ一言だけ確認する
2. Notion「商品企画フィードバックログ」に1行追記する（形式は notion-rules.md 参照。ページが無ければ作成）
3. 追記した行をユーザーに見せて確認する

## 分析モード
ログ全件を読み、以下を報告する:
1. 没・見送り理由のパターンTOP5（頻度順、実際の理由の引用付き）
2. 採用・ヒットに共通する特徴（フックの型、距離、アンカーの種類など）
3. 「次のアイデア生成で変えるべきこと」の提言3つ
4. 発売後の結果（ヒット/不発）が未記録の採用案があればリマインドする

日本語で回答。
