# 新商品企画 提案テンプレ（企画会議・第1ゲート用）

proposal-audit スキルの出力仕様書。項目・順序・各項目の目的を変えないこと。
会議運営パート（後半）は提案書出力には含めないが、賛否の裁定基準として必ず読むこと。

> **この会議で決めること**: この案が「マーケ検討に進む価値があるか」。ヒットするかどうかはまだ誰にも分からない。判定材料は意見ではなく証拠。
> **分量目安**: Notion 1ページ・会議15分で監査できる量。
> **鉄則**: 全ての主張に出典（URL・レビュー原文・スクショ）。確認できないものは「未確認」と正直に書く。未確認が多いのは減点ではない — 未確認を確認済みのように書くのが失格。

## 0. 提案サマリー（3行）
`【誰の】＿＿な悩みを【何で】＿＿によって解決し【いくらで】¥＿＿で売る。`
- **目的**: 最初の30秒で全員の頭を揃える。
- **会議で見る**: 3行で書けない案は、まだ企画として結晶していない。

## 1. フックとFVコピー（広告の顔）
- フック（広告コピー30文字以内）／LPのFVコピー
- **目的**: このモデルでは商品＝広告クリエイティブ。**FVが書けない商品は企画として存在しない**のと同じ。
- **注意**: 機能の羅列ではなく「スクロールが止まる一撃」。複数案あってよい。
- **会議で見る**: 自分のInstagramフィードにこれが流れてきて、指が止まるか。

## 2. 納得の理屈（1文）
`＿＿だから＿＿になる（例: 温めると血行が良くなり首が緩む）`
- **目的**: 衝動買いは「刺さる」＋「腑に落ちる」の2段。腑に落ちる部分の検品。
- **注意**: **1文で書けなければ飛びすぎ**。説明に3段落必要な理屈は広告では機能しない。

## 3. 悩みアンカーと証拠
- 建前（口に出す悩み）／本音（本当の感情）
- 証拠: 低評価レビュー原文の引用 **3件以上**（URL付き）／SNS・知恵袋の生声／統計（出典URL必須）
- **目的**: アイデアを「実在する悩み」に着地させる。会議中に案が変異したとき、ここに戻って着地し直すための錨。
- **注意**: 「自分も欲しいと思った」はn=1であり証拠ではない（きっかけとしては書いてよい）。
- **会議で見る**: 引用は文脈通りか。都合のいい切り取りをしていないか。

## 4. 需要の実測シグナル（カネが回っている証拠）
- 配信中の競合・類似訴求の広告（**配信開始日**必須。3ヶ月以上継続＝経済性成立の最強の証拠）
- 競合のレビュー増加速度 → 推定月販／クラファン・ランキング実績
- **目的**: 机上の市場規模データより「誰かが広告費を払い続けている」という観測事実を優先する。
- **会議で見る**: シグナルがゼロのカテゴリなら「なぜ誰も広告を回していないのか」を6番で説明できているか。

## 5. 競合（顧客同一性で選ぶ）＋ ¥3,000類似品チェック
- 競合3商品以上: URL／価格／機能／それでもうちが選ばれる理由（表形式）
- Amazon/楽天で**実際に検索した結果**: ¥3,000以下の類似品が上位に「ある／ない」。あるなら、その低評価レビューにある攻め筋
- **発売2年以内の新規参入チェック**: プレスリリース・ニュース検索で直近参入した大手・新商品を確認（検索順位もレビュー数もまだ低く、通常の競合調査から漏れやすい）
- **組み合わせ攻撃チェック**: 既存品A＋既存品Bの組み合わせで同じ便益がより安く手に入らないか（例: 安価タオル枕¥3,000＋後付けヒーター¥7,700）
- **注意**: 競合は「見た目・素材が似ている商品」ではなく「**同じ人が代わりに買う物**」で選ぶ。サプリ・施術・アプリも競合になり得る。¥0の代替（家にある物での自作）も含める。
- **会議で見る**: 競合欄が空欄・雛形のままの提案は監査不能なので差し戻し。

## 6. 空白の理由
`この商品がまだ世に無い（or 消えた）のはなぜか`
- 過去に類似品が出て消えた形跡のWeb検索結果: あり（何が起きたか）／なし。**特に「大手が一度出して引っ込めた」形跡**は最重要。
- **目的**: 「思いつかれていないだけ」は世界で最も疑わしい仮説。空白の理由が「作るのが難しいから」なら参入障壁、「需要がないから」なら死刑宣告。
- **会議で見る**: 先行品が消えた形跡が出たら、**理由を調べるまで次工程に進めない**。

## 7. 価格と経済性（粗くてよい、前提は明記）
- 想定販売価格（無難な価格ではなく**支払い意思の上限**を狙う）＋根拠3点: ①代替手段の累計コスト ②競合の最高価格帯と売れ行き ③悩みの深さ
- 原価率30%で仕入れ¥＿＿は現実的か（自社の既存部材・実績の流用可否も）
- 悲観シナリオ（CPC¥150×CVR1%）でCPAは粗利（売価×70%）に収まるか。標準シナリオ（CPC¥100×CVR1.5%）での広告費率も併記
- **注意**: 概算でよいが**根拠のない価格を単体で書かない**。想定価格がカテゴリ上限を超える場合はその旨を明記。
- **会議で見る**: 価格が「なんとなく競合と同じ帯」に丸まっていないか。

## 8. セルフキル表（提案者が自分で殺しに行った結果）
| キル軸 | 判定（通過/グレー/重傷/即死） | 証拠・根拠 |
|---|---|---|
| ①価格競合（¥3,000類似品・組み合わせ攻撃） | | |
| ②法規制（薬機法・景表法でフックが言えるか） | | |
| ③経済性（原価・物流・悲観CPA） | | |
| ④空白の理由 | | |
| ⑤本音/建前ギャップ（買う瞬間・届く瞬間の恥ずかしさ） | | |
| ⑥ターゲット量（¥10,000払う人が年3,000万円分いるか） | | |
- **目的**: 会議を感想戦ではなく「この表の証拠は本物か」の監査にする。
- **注意**: **全軸「通過」の提案はむしろ疑う**（本気で殺しに行っていない証拠）。グレーが1〜3個あるのが正直な提案。
- **会議で見る**: 判定の正しさではなく証拠の質を見る。グレーは没理由にしない — 次工程の宿題にする。

## 9. 3秒映像案
`リール/TikTokの最初の3秒で見せる画`
- **目的**: 効果・使用感が3秒で映像化できない商品は、このビジネスモデルでは広告が作れない。

## 10. 検証課題の申告（この案が死ぬとしたら）
- 提案者が考える最大の死因／マーケ検討で調べてほしいこと（**安く検証できる順**）
- **目的**: グレーを隠さず申告した提案者が得をする設計にする。次工程の調査設計書を兼ねる。

# 会議の回し方（賛否裁定の基準。出力には含めないが裁定時に従う）

**事前**
- 提案書は前日までに共有。会議での初見プレゼンは禁止。
- 参加者は最低限、3番のレビュー原文と4番の広告リンクを**生で**見てくる（要約ではなく）。

**進行（15〜20分/案）**
1. 監査: 証拠の質を突く（「その引用は文脈通り？」「配信期間は確認した？」）
2. 死因指名（1人1つ・必須）: **◯△✖️投票は廃止**。各自「この案が死ぬとしたら何で死ぬか」を証拠または検証方法付きで1つ。賛成意見は言わなくてよい。
3. 提案者の応答（反証 or「グレーとして受領」）
4. 判定。

**判定は3択（2択にしない）**
- **進める**: マーケ検討へ。持ち越す検証課題を明記する。
- **差し戻し**: 足りない証拠を**具体的に指名**して再提出可（例:「先行品が消えた理由を調べてから」）。一番多くなるべき判定はこれ。
- **没**: どのキル軸で死んだかを記録し、死因を制約条件に変換する（「次の案は◯◯を避ける」）。感想では殺さない。

**最終判断者のセルフチェック — 判定を口にする前に**
- [ ] 一次証拠（レビュー原文・広告）を自分の目で見たか。要約と発表だけで判断していないか
- [ ] いま判断しているのは「ヒットするか」ではなく「**次の調査コストに値するか**」か
- [ ] 会議中に案が変異していないか（変異したら、その場で採否を決めず差し戻して3番のアンカーに着地し直させる）
- [ ] 死因候補のうち**最も安く検証できるもの**はどれか。それを検証課題に指定したか
- [ ] 即席の判断基準を発明していないか（発明したら、その場で採用せず後日テンプレの基準に昇格させるか捨てるか決める）

**記録（会議の最後に2分）**
- 全員の死因指摘・判定・**各自のヒット確率◯%と理由一言**をフィードバックログへ。発売後の結果と突き合わせ、半年ごとに「誰のどの直感が当たるか」を振り返る。
